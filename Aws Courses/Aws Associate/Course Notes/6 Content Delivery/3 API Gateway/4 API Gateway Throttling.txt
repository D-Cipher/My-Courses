API Gateway Throttling

API Gateway Throttling -> API gateway allows throttle at multiple levels including global and by service call
- Throttling requests to prevent attacks
- Throttling to protect backend services from being overloaded
- Client SDKs generated by API Gateway retries calls automatically
- Configured at multiple levels including Global and Service Call
- 429 HTTP response "Too Many Requests" error


Throttling Limit and Burst Capacity:
- Default throttling limit of 10K requests per second
- Additional 5K burst capacity available past the 10K
